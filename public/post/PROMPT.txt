This is an astro project. 
When I use `npm run build`, 
First it run @entriesGenerator.js 
entriesGenerator.js is to scan files recursively in /src/post. 
For the files in /src/post/archives ignore it. 
For the files in /src/post/file, if the fileName starts with [YYYY-MM-DD] this format, then extract the date and the string after, to write post' title and it's date into @entries.js. Else, write post' title and it's file updated date as date into @entries.js.
For the files in /src/post/html, first inject inject.css into all the html files in that folder and move the injected version into public/post. And if the fileName starts with [YYYY-MM-DD] this format, then extract the date and the string after, to write post' title and it's date into @entries.js. Else, write post's title and it's file updated date as date into @entries.js.
For the files in /src/post/qmd, run script like this quarto render fileName.qmd --to html but the destination should be public/post. And if the fileName starts with [YYYY-MM-DD] this format, then extract the date and the string after, to write post' title and it's date into @entries.js. Else, write post' title and it's file updated date as date into @entries.js.
And copy the files in /src/post/file to /public/posts. For the pin folders do the same thing if it have files.
Finally, we get all the post title and date in entries.js and all the files in the right form ready in /public/post or /public/post/pin
